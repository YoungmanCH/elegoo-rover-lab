# elegoo-rover-lab

ELEGOO Smart Robot Car Kit V4.0 を使って、ロボットカーの基礎を一から学ぶための実験リポジトリです。

まずは「組み立てて動かす」だけではなく、モーター制御、センサー入力、簡単な自律走行、そして将来的には Raspberry Pi や ROS 2 への拡張までを少しずつ試していきます。

## 目的

このリポジトリの目的は、ロボットの基本構成を手を動かしながら理解することです。

* 電池でロボットに電力を供給する
* Arduino UNO で命令を出す
* モータードライバでモーターを動かす
* 4つのタイヤを使って前進・後退・旋回する
* 超音波センサーで障害物との距離を測る
* ライントラッキングモジュールで黒線を検出する
* サーボモーターでセンサーの向きを変える
* 自分でコードを書き換えて挙動を変える

## 使用キット

* ELEGOO Smart Robot Car Kit V4.0

### Arduino UNO R3

Arduino UNO R3 は、このロボットの中心になる制御基板です。

普通のパソコンのCPUほど高性能ではありませんが、小さなマイコンが載っていて、プログラムを実行できます。

このロボットでは、主に以下のようなことを行います。

* モーターを動かす命令を出す
* 超音波センサーの値を読む
* ライントラッキングモジュールの値を読む
* サーボモーターの角度を変える
* スマホやカメラ側から来た命令に応じて動く

イメージとしては、Arduino UNO はロボットの「脳」です。

Arduino UNO = 命令を考えて出す部分

### Expansion Board

Expansion Board は、Arduino UNO の上に取り付ける追加の基板です。

Arduino UNOだけでもセンサーやモーターをつなぐことはできますが、ピンが多くて配線が複雑になりやすいです。

そこで、Expansion Board を使うことで、モーター、センサー、サーボ、電源などをつなぎやすくしています。

Arduino UNO = 脳
Expansion Board = 配線を整理するための追加ボード

組み立てでは、Arduino UNO の上に Expansion Board をまっすぐ差し込みます。
ピンがずれていないか、向きが合っているかを確認してから取り付けます。

### TB6612 Motor Driver Chip

TB6612 Motor Driver Chip は、モーターを動かすための重要な部品です。

Arduino UNO は命令を出すことはできますが、DCモーターを直接動かすほど大きな電流を流すことはできません。
そのため、Arduino UNO と DCモーターの間にモータードライバが入ります。

Arduino UNO
↓ 命令
TB6612 Motor Driver Chip
↓ 強い電流
DCモーター
↓
タイヤが回る

TB6612 は、モーターに流す電気の向きや強さを変えることで、以下のような制御を行います。

* 前に回す
* 後ろに回す
* 止める
* 速さを変える

つまり、TB6612 Motor Driver Chip は、Arduinoからの小さな命令を受け取って、モーターを実際に動かすための「力を出す係」です。

### DC Motor

DC Motor は、電気を流すと回転する部品です。

このロボットでは、DC Motor がタイヤを回すことで車体が動きます。

電気
↓
DC Motor が回る
↓
タイヤが回る
↓
ロボットが進む

ELEGOO Smart Robot Car Kit V4.0 では、4つのタイヤを使ってロボットを動かします。

基本的な考え方は、左側のタイヤと右側のタイヤを別々に動かすことです。

左側のタイヤ + 右側のタイヤ

たとえば、左右どちらも前に回せば前進します。
左側と右側の動かし方を変えると、右折や左折ができます。

### Tire / Wheel

Tire / Wheel は、モーターの回転を地面に伝えて、ロボットを動かす部品です。

このキットは、車のように前輪の角度を変えて曲がるタイプではありません。
左右のタイヤの回転差で曲がります。

* 左右どちらも前回転 → 前進
* 左右どちらも後回転 → 後退
* 左が前回転、右が停止 → 右に曲がる
* 右が前回転、左が停止 → 左に曲がる
* 左が前回転、右が後回転 → その場で右回転

### Cellbox

Cellbox は電池ボックスです。

ロボットに必要な電力を供給します。

電池
↓
Cellbox
↓
ロボット全体に電力を供給

配線するときは、プラスとマイナスを間違えないようにします。

多くの場合、以下のように考えます。

* 赤い線 = プラス
* 黒い線 = マイナス / GND

### Ultrasonic Sensor Module

Ultrasonic Sensor Module は、前方の障害物までの距離を測るためのセンサーです。

日本語では、超音波センサーモジュールと呼ばれます。

このセンサーは、人間には聞こえない高い音を出し、その音が物に当たって戻ってくるまでの時間を測ります。

超音波を出す
↓
障害物に当たる
↓
音が戻ってくる
↓
距離を計算する

このセンサーを使うと、以下のような動きが作れます。

* 前に物があったら止まる
* 障害物が近づいたら後退する
* 左右を見て空いている方へ曲がる

### SG90 Servo

SG90 Servo は、小型のサーボモーターです。

普通のDCモーターは、電気を流すとくるくる回り続けます。
一方、サーボモーターは「指定した角度に動く」モーターです。

* 0度に向ける
* 90度に向ける
* 180度に向ける

このキットでは、SG90 Servo に超音波センサーを取り付けることで、センサーの向きを変えられます。

* 左を見る
* 前を見る
* 右を見る

これにより、ロボットは前だけでなく、左右の障害物も確認できます。

### Ultrasonic Sensor Module Holder

Ultrasonic Sensor Module Holder は、超音波センサーを固定するためのホルダーです。

SG90 Servo と組み合わせることで、超音波センサーを左右に動かせるようになります。

超音波センサー
↓
Holder
↓
SG90 Servo
↓
左右に向きを変える

人間にたとえると、超音波センサーが「目」、ホルダーが「顔」、SG90 Servo が「首を動かす筋肉」のような役割です。

### Line Tracking Module

Line Tracking Module は、床の黒い線を検出するためのセンサーです。

日本語では、ライントラッキングモジュール、またはライン追跡センサーと呼ばれます。

このセンサーは、床に光を当てて、その反射の違いを読み取ります。

* 白い床 → 光をよく反射する
* 黒い線 → 光をあまり反射しない

これを使うと、ロボットを黒い線に沿って走らせることができます。

* 中央のセンサーが黒を検出 → 直進
* 左のセンサーが黒を検出 → 左に補正
* 右のセンサーが黒を検出 → 右に補正

### MPU6050 Gyro Module / GY-521 Module

MPU6050 Gyro Module は、ロボットの傾き・加速度・回転を検出するためのセンサーです。

キット内では `MPU6050 gyro module` や `GY-521 Module` のように書かれていることがあります。
GY-521 は、MPU6050 というセンサーを小さな基板に載せたモジュール名として扱われることが多いです。

このモジュールでは、主に以下の2つを測れます。

* 加速度
* 角速度

加速度は、ロボットが動き出したり、止まったり、傾いたりしたときの変化です。
角速度は、ロボットがどれくらいの速さで回転しているかを表します。

加速度 = 動き出し・停止・傾きの変化
角速度 = 回転の速さ

人間にたとえると、ロボットの「バランス感覚」に近い部品です。

このセンサーを使うと、以下のような情報を得ることができます。

* ロボットが傾いた
* ロボットが回転した
* ロボットが揺れた
* 急に動いた
* 旋回しているかどうか

最初は深く使わなくてもよいですが、将来的にロボットの姿勢や動きを知るために役立ちます。

### ESP32-CAM

ESP32-CAM は、カメラとWi-Fi機能を持った小さなモジュールです。

このキットでは、スマホアプリからカメラ映像を見たり、ロボットを操作したりするために使われます。

スマホ
↓ Wi-Fi
ESP32-CAM
↓
Arduino UNO
↓
ロボットが動く

最初は、ESP32-CAM の中身を細かく理解しなくても大丈夫です。
まずは Arduino UNO 側でモーター制御を理解し、その後でカメラやスマホ連携を見ていくと分かりやすいです。

### Jumper Wires

Jumper Wires は、基板やセンサー同士をつなぐための線です。

ロボットでは、Arduino UNO、Expansion Board、センサー、モーター、電源などを接続するために使います。

部品A
↓
Jumper Wire
↓
部品B

ジャンパーワイヤには、オス・メスの種類があります。

* オス = ピンが出ている
* メス = 穴になっている

接続するときは、説明書のピン番号や向きを確認してから挿します。

### Screws / Nuts / Standoffs

Screws、Nuts、Standoffs は、部品を車体に固定するためのネジ類です。

* Screw = ネジ
* Nut = ナット
* Standoff = スペーサー / 支柱

Standoff は、基板をシャーシから少し浮かせて固定するために使います。

基板を直接シャーシに触れさせると、ショートしたり、配線しにくくなったりすることがあります。
そのため、Standoff を使って少し高さを出します。

### Chassis

Chassis は、ロボットの車体部分です。

モーター、タイヤ、Arduino UNO、Expansion Board、電池ボックス、センサーなどを取り付ける土台になります。

Chassis = ロボットの体

### USB Cable

USB Cable は、PCとArduino UNOをつなぐために使います。

主な用途は、Arduino IDEで書いたプログラムをArduino UNOに書き込むことです。

PC
↓ USB Cable
Arduino UNO
↓
ロボットの動きが変わる

スマホ操作で動かすだけならUSBケーブルを使わない場合もありますが、自分でコードを書き換えるときには必要になります。


## 学習ロードマップ

### Step 1: 公式コードを動かす

まずは公式の説明書どおりに組み立て、サンプルコードでロボットが動くことを確認します。

* 前進
* 後退
* 左折
* 右折
* 停止
* スマホ操作

### Step 2: モーター制御を理解する

最初に見るべきなのは、モーターを動かしているコードです。

やること:

* 前進する
* 停止する
* 右に曲がる
* 左に曲がる
* 回転時間を変える
* モーター速度を変える

例:

```text
3秒前進
1秒停止
1秒右回転
3秒前進
停止
```

### Step 3: 超音波センサーを使う

前方の障害物までの距離を測ります。

やること:

* 距離をシリアルモニタに表示する
* 20cm以内に障害物があったら止まる
* 障害物があったら左右を確認する
* 空いている方向へ曲がる

### Step 4: ライントラッキングを試す

床に黒いテープを貼って、その線に沿って走らせます。

やること:

* 左・中央・右のセンサー値を読む
* 中央が黒なら直進する
* 左が黒なら左に補正する
* 右が黒なら右に補正する

### Step 5: 自分の制御ロジックを書く

公式コードをそのまま使うだけではなく、自分で動きのルールを作ります。

例:

* 3秒進んで右折する
* 障害物があったら停止する
* 障害物を避けて進む
* 黒線に沿って走る
* 一定時間ごとに周囲を確認する

### Step 6: 将来の拡張

余裕が出てきたら、以下にも挑戦します。

* Raspberry Pi から Arduino UNO を操作する
* PC からシリアル通信で命令を送る
* カメラ映像を使った操作
* LiDAR の追加
* ROS 2 との連携
* `/cmd_vel` でロボットを動かす

## 最初の実験

### 001: 前進・停止・旋回

最初の目標は、ロボットに決まった動きをさせることです。

```text
1. 3秒前進する
2. 1秒停止する
3. 1秒右に回る
4. 3秒前進する
5. 停止する
```

この実験で学ぶこと:

* モーターの回し方
* 左右のタイヤの違い
* 曲がる仕組み
* 速度と時間の関係
* 実機は思ったよりまっすぐ進まないこと

## メモ

このリポジトリでは、きれいな完成品を作ることよりも、実際に動かして失敗しながら理解することを重視します。

ロボットは、コードだけではなく、電池、モーター、配線、床、タイヤのすべり方などにも影響されます。
そのため、実験結果もなるべく記録していきます。

# Demo1: TB6612 & MPU6050 のコード解説

公式キット `02SmartRobotCarV4.0_Move/TB6612 & MPU6050/Demo1` のソースコードを読み解くためのドキュメントです。

Arduino の基礎（`setup()` / `loop()`）については [arduino-basics.md](arduino-basics.md) を参照してください。

---

## ファイル構成と拡張子

```
Demo1/
├── Demo1.ino                         ← メインファイル（setup / loop がある）
├── DeviceDriverSet_xxx0.h            ← ヘッダファイル（設計図・宣言）
├── DeviceDriverSet_xxx0.cpp          ← 実装ファイル（設計図の中身）
└── ApplicationFunctionSet_xxx0.cpp   ← 動きの制御ロジック
```

### 拡張子の意味

| 拡張子 | 意味 |
|--------|------|
| `.ino` | Arduino スケッチ。Arduino IDE が最初に読むファイル。`setup()` と `loop()` がここにある |
| `.h`   | **ヘッダファイル**。「こういうクラスや関数がありますよ」という**宣言（設計図）**だけを書く |
| `.cpp` | **実装ファイル**。`.h` で宣言した関数の**中身（実際の処理）**を書く |

### なぜ .h と .cpp に分けるのか

C++ では「宣言」と「実装」を分けるのが基本的な作法です。

```
.h  = メニュー表（何ができるかの一覧）
.cpp = 厨房（実際の調理方法）
```

他のファイルは `.h` だけを `#include` すれば、中身の実装を知らなくても使えます。

## C++ の基本概念

プログラミングの基礎はあるが C++ に疎い人向けに、このコードで登場する概念をまとめます。

### `#include` = import

```cpp
#include "DeviceDriverSet_xxx0.h"   // 自作ファイルの読み込み（" " を使う）
#include <Arduino.h>                // ライブラリの読み込み（< > を使う）
```

Python の `import` と同じ役割です。

- `" "` → 同じフォルダにある自作ファイル
- `< >` → システムやライブラリのファイル

### `#define` = 定数の置き換え

```cpp
#define PIN_Motor_PWMA 5
#define direction_just true
#define speed_Max 255
```

コンパイル前に「文字をそのまま置き換える」マクロです。
`PIN_Motor_PWMA` と書いたところは全部 `5` に置き換わります。

Python でいう `PIN_MOTOR_PWMA = 5` の定数宣言に近いです。

### `class` = クラス（設計図）

```cpp
class DeviceDriverSet_Motor {
public:        // ← 外から使える部分
  void DeviceDriverSet_Motor_Init(void);
  void DeviceDriverSet_Motor_control(...);
private:       // ← クラスの中だけで使う部分
  #define PIN_Motor_PWMA 5
};
```

- `public:` → 外のコードから呼べる関数や変数
- `private:` → クラスの内部だけで使う（外から触れない）

### インスタンス化（実体を作る）

```cpp
DeviceDriverSet_Motor AppMotor;
```

クラス（設計図）から実体を作ることを「インスタンス化」と呼びます。
Python でいう `app_motor = DeviceDriverSet_Motor()` と同じです。

C++ では `()` を省略して書くことが多いです（引数なしのコンストラクタの場合）。

### `::` = スコープ解決演算子（所属を示す）

```cpp
void DeviceDriverSet_Motor::DeviceDriverSet_Motor_Init(void) {
  // 実装の中身
}
```

「`DeviceDriverSet_Motor` クラスに属する `DeviceDriverSet_Motor_Init` 関数」という意味です。

`.h` で宣言した関数の中身を `.cpp` に書くとき、この書き方で「どのクラスの関数か」を示します。

### `enum` = 列挙型（選択肢リスト）

```cpp
enum SmartRobotCarMotionControl {
  Forward,       // = 0
  Backward,      // = 1
  Left,          // = 2
  Right,         // = 3
  LeftForward,   // = 4
  LeftBackward,  // = 5
  RightForward,  // = 6
  RightBackward, // = 7
  stop_it        // = 8
};
```

自動的に `0, 1, 2, 3...` の番号が割り振られます。
Python の `Enum` と同じ考え方で、「前進は0番、後退は1番...」を名前付きで管理するものです。

マジックナンバー（意味不明な数字）を避けるために使います。
`3` と書くより `Right` と書いた方が読みやすいです。

### `struct` = データをまとめる箱

```cpp
struct Application_xxx {
  SmartRobotCarMotionControl Motion_Control;
};
```

Python の `dataclass` に近いです。
関連するデータをひとまとめにするだけの軽い構造体です。

`class` との違い：`struct` はデフォルトで全部 `public`、`class` はデフォルトで全部 `private` です。
データだけ持たせたい場合は `struct`、関数も持たせたい場合は `class` を使うことが多いです。

### `extern` = 別ファイルの変数を使う宣言

```cpp
extern DeviceDriverSet_Motor AppMotor;
```

「`AppMotor` は他のファイル（Demo1.ino）で作られているけど、このファイルからも使うよ」という宣言です。

実体を作るのではなく、「他で作られた実体を参照する」ためのものです。

### `void` = 戻り値なし / 引数なし

```cpp
void DeviceDriverSet_Motor_Init(void);
```

- 先頭の `void` → この関数は値を返さない
- 引数の `(void)` → この関数は引数を受け取らない（C++ では `()` と同じ意味）

### `uint8_t` / `boolean` = 型名

| 型 | 意味 | 範囲 |
|---|---|---|
| `uint8_t` | 符号なし8ビット整数 | 0〜255 |
| `boolean` | 真偽値 | `true` / `false` |

`uint8_t` は速度（0〜255）やピン番号に使われます。
Arduino のアナログ出力（PWM）が 0〜255 の範囲なので、ちょうど `uint8_t` に収まります。

### `static` = ファイル内だけで使える

```cpp
static void ApplicationFunctionSet_SmartRobotCarMotionControl(...) { ... }
```

`static` が付いた関数やグローバル変数は、そのファイルの中でしか使えません。
他のファイルから直接呼ばれることを防ぎます。

### `switch / case` = 分岐

```cpp
switch (direction) {
case Forward:
  // 前進の処理
  break;
case Backward:
  // 後退の処理
  break;
default:
  // どれにも当てはまらない場合
  break;
}
```

Python の `match / case`（3.10以降）や `if / elif` の連鎖に相当します。
`break` を忘れると次の `case` にそのまま落ちてしまうので注意が必要です。

### `#ifndef` / `#define` / `#endif` = インクルードガード

```cpp
#ifndef _DeviceDriverSet_xxx0_H_
#define _DeviceDriverSet_xxx0_H_

// ヘッダの中身

#endif
```

同じ `.h` ファイルが複数回 `#include` されても、中身が二重に読み込まれないようにする仕組みです。
C++ のヘッダファイルでは、ほぼ必ずこれを書きます。

## Demo1.ino の詳細解説

Demo1.ino はこのプロジェクトのメインファイルです。

### 全体のコード

```cpp
#include <avr/wdt.h>
#include "DeviceDriverSet_xxx0.h"
#include "ApplicationFunctionSet_xxx0.cpp"

DeviceDriverSet_Motor AppMotor;
Application_xxx Application_SmartRobotCarxxx0;

void setup() {
  AppMotor.DeviceDriverSet_Motor_Init();
  delay(2000);
  for (Application_SmartRobotCarxxx0.Motion_Control = 0;
       Application_SmartRobotCarxxx0.Motion_Control < 9;
       Application_SmartRobotCarxxx0.Motion_Control = Application_SmartRobotCarxxx0.Motion_Control + 1)
  {
    ApplicationFunctionSet_SmartRobotCarMotionControl(
      Application_SmartRobotCarxxx0.Motion_Control, 200);
    delay(1000);
  }
}

void loop() {
}
```

### 1行ずつの解説

#### インクルード部分

```cpp
#include <avr/wdt.h>                        // ウォッチドッグタイマー（このデモでは未使用）
#include "DeviceDriverSet_xxx0.h"           // モータークラスの設計図を読み込む
#include "ApplicationFunctionSet_xxx0.cpp"  // 動き制御の関数を読み込む
```

`avr/wdt.h` はウォッチドッグタイマー（プログラムが固まったときに自動リセットする機能）のライブラリですが、このデモでは使われていません。

#### インスタンス生成

```cpp
DeviceDriverSet_Motor AppMotor;                // モータークラスの実体を作る
Application_xxx Application_SmartRobotCarxxx0; // 動き管理用のデータ構造を作る
```

ここで「設計図（class / struct）」から「実物（インスタンス）」を作っています。

#### setup()

```cpp
AppMotor.DeviceDriverSet_Motor_Init();   // モーターのピンを初期化
delay(2000);                             // 2秒待つ（起動直後の安定待ち）
```

`DeviceDriverSet_Motor_Init()` の中身は `DeviceDriverSet_xxx0.cpp` にあり、モーター制御に使う5本のピンを `OUTPUT` に設定しています。

#### for ループ

```cpp
for (Application_SmartRobotCarxxx0.Motion_Control = 0;
     Application_SmartRobotCarxxx0.Motion_Control < 9;
     Application_SmartRobotCarxxx0.Motion_Control = Application_SmartRobotCarxxx0.Motion_Control + 1)
```

`Motion_Control` を 0 から 8 まで 1 ずつ増やしながらループします。
これは enum の `Forward(0)` から `stop_it(8)` に対応しています。

```cpp
ApplicationFunctionSet_SmartRobotCarMotionControl(
  Application_SmartRobotCarxxx0.Motion_Control,  // 方向（0〜8）
  200                                            // 速度（0〜255）
);
delay(1000);  // 各動作を1秒ずつ実行
```

つまり、9つの動きを1秒ずつ順番に実行するデモです。

#### 実行される動きの順番

| Motion_Control | enum名 | 動き | 時間 |
|---|---|---|---|
| 0 | Forward | 前進 | 1秒 |
| 1 | Backward | 後退 | 1秒 |
| 2 | Left | 左旋回（その場回転） | 1秒 |
| 3 | Right | 右旋回（その場回転） | 1秒 |
| 4 | LeftForward | 左前方カーブ | 1秒 |
| 5 | LeftBackward | 左後方カーブ | 1秒 |
| 6 | RightForward | 右前方カーブ | 1秒 |
| 7 | RightBackward | 右後方カーブ | 1秒 |
| 8 | stop_it | 停止 | 1秒 |

#### loop() は空

```cpp
void loop() {
}
```

このデモでは `setup()` で全動作を実行して終わりです。
`loop()` には何も書かれていないので、それ以降ロボットは何もしません。

## モーター制御の仕組み

### ハードウェアのピン割り当て

`DeviceDriverSet_xxx0.h` で定義されているピン番号です。

| 定数名 | ピン番号 | 役割 |
|---|---|---|
| `PIN_Motor_PWMA` | 5 | モーターA（右）の速度制御（PWM） |
| `PIN_Motor_PWMB` | 6 | モーターB（左）の速度制御（PWM） |
| `PIN_Motor_AIN_1` | 7 | モーターA（右）の方向制御 |
| `PIN_Motor_BIN_1` | 8 | モーターB（左）の方向制御 |
| `PIN_Motor_STBY` | 3 | スタンバイ（HIGH で有効化、LOW で停止） |

### Arduino の出力関数

| 関数 | 出力 | 用途 |
|---|---|---|
| `digitalWrite(pin, HIGH/LOW)` | ON / OFF（デジタル） | 方向制御、スタンバイ制御 |
| `analogWrite(pin, 0〜255)` | 段階的な電圧（PWM） | 速度制御 |

### 方向と速度の制御方法

TB6612 モータードライバは、ピンの HIGH / LOW の組み合わせでモーターの方向を制御します。

| 操作 | AIN_1 / BIN_1 | PWM | STBY |
|---|---|---|---|
| 正転 | HIGH | 速度値（0〜255） | HIGH |
| 逆転 | LOW | 速度値（0〜255） | HIGH |
| 停止 | - | 0 | LOW |

### モーターA・B と動きの関係

```
モーターA = 右側タイヤ
モーターB = 左側タイヤ
```

| 動き | A（右）方向 | B（左）方向 | A速度 | B速度 |
|---|---|---|---|---|
| Forward | 正転 | 正転 | 200 | 200 |
| Backward | 逆転 | 逆転 | 200 | 200 |
| Left | 正転 | **逆転** | 200 | 200 |
| Right | **逆転** | 正転 | 200 | 200 |
| LeftForward | 正転 | 正転 | 200 | **100**（半分） |
| LeftBackward | 逆転 | 逆転 | 200 | **100**（半分） |
| RightForward | 正転 | 正転 | **100**（半分） | 200 |
| RightBackward | 逆転 | 逆転 | **100**（半分） | 200 |
| stop_it | void | void | 0 | 0 |

- Left / Right: 左右のタイヤを逆方向に回して**その場で回転**する
- LeftForward / RightForward: 片側のタイヤを遅くして**ゆるやかにカーブ**する

## ファイル間の全体の流れ

```
Demo1.ino（メイン）
  │
  ├── #include "DeviceDriverSet_xxx0.h"  ← クラスの宣言を読む
  │      └── DeviceDriverSet_xxx0.cpp    ← その実装（ピン制御の中身）
  │
  ├── #include "ApplicationFunctionSet_xxx0.cpp" ← 動き制御関数を読む
  │      └── enum / struct / switch文で方向→モーター指示に変換
  │
  └── setup()
       └── for文で 0〜8 を回す
            └── ApplicationFunctionSet_SmartRobotCarMotionControl()
                 └── AppMotor.DeviceDriverSet_Motor_control()
                      └── digitalWrite / analogWrite でピンを操作
                           └── TB6612 → モーター → タイヤが動く
```

### 処理の流れを言葉で表すと

1. `Demo1.ino` の `setup()` が呼ばれる
2. `DeviceDriverSet_Motor_Init()` で5本のピンを OUTPUT に設定する
3. 2秒待つ
4. for ループで `Motion_Control` を 0〜8 まで回す
5. 各値に対して `ApplicationFunctionSet_SmartRobotCarMotionControl()` を呼ぶ
6. この関数の中の `switch` 文で、方向に応じたモーター制御パラメータを決める
7. `DeviceDriverSet_Motor_control()` が実際に `digitalWrite` / `analogWrite` でピンを操作する
8. TB6612 がピンの信号を受けてモーターに電流を流す
9. タイヤが回る
10. 1秒待って次の動きへ

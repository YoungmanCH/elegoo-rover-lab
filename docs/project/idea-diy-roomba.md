# アイディアメモ: 自作ルンバ構想（Webコックピット付き）

> ステータス: **アイディア段階（余談）**。ハード仕様の確定情報は [machine-reference.md](../reference/machine-reference.md) を参照。
> ここは「やりたいこと」と「どこが簡単/難しいか」をざっくり残すための走り書き。

## 1. コンセプト

ELEGOO Smart Robot Car V4.0 の先端〜下部に **クイックルワイパー（ドライシート）** を付けて、簡易お掃除ロボット（自作ルンバ）にする。
それを **Webから操作・掃除を設定**でき、部屋マップ付きの「コックピット画面」で見られるシステム。

## 2. 方針（温度感）

- 掃除レベルは **(A)「部屋全体をいい感じに」** で割り切る。きっちりした範囲カバーは狙わない。
- マッピングは **スマホで部屋をスキャン**して、ダッシュボードの背景（舞台）として使う。
- 位置の正確さは「今はざっくり、将来拡張」と割り切る。

## 3. できること / 難しいこと

| 要素 | 実現性 | メモ |
|---|---|---|
| Web操作（起動/停止/速度） | ◎ | 既存JSONプロトコル（§9）をESP32経由で送るだけ |
| ランダム掃除（壁回避） | ◎ | 初代ルンバ方式。既存の障害物回避モード（超音波）がほぼ流用可 |
| ライブ計器（電圧/距離/向き） | ◎ | N=21/22/23 ＋ 電圧A3 ＋ IMU Yaw を画面に流す |
| 部屋スキャン地図の表示 | ○ | iPhone/iPad Pro の LiDAR＋RoomPlan で間取り取得 → 背景に敷く |
| 地図上の「本物の現在地」 | △ | **自己位置推定が無い**ので今はざっくり。下記の限界参照 |
| きっちり範囲カバー（ジグザグ） | △ | 位置推定が前提。(A)では狙わない |

### 限界（正直なところ）
- 車輪エンコーダが無く、IMUのYaw（向き）しか持たない → **位置は積分誤差で数十秒でズレる**。
- だからスキャンした地図は「ナビ用」ではなく **演出（デジタルツインの舞台）／広さの目安／立入禁止エリアのUI** としての価値が主。
- 地図上の点を“本物の位置”にするには将来 **天井カメラ＋ARマーカー(ArUco)** か **エンコーダ追加** が必要（Phase 3）。

## 4. 機構メモ（クイックル）
- 前方より **車体下〜後方に引きずる**方が掃除効果は高い（前付けはゴミを押し出すだけになりがち）。
- 小型軽量で押し付け圧が足りず素通りしがち → 軽いバネ/オモリでダウンフォースを。
- シート摩擦がモータ負荷になる → **フローリング限定・低速**が無難。カーペットは厳しい。

## 5. 段階ロードマップ

```
Phase 1 : ランダム掃除 + Web操作 + ライブ計器        ← 今の構成で動く（コア）
Phase 2 : スマホで部屋スキャン → 地図を背景に表示      ← 見栄え担当（デジタルツイン演出）
Phase 3 : 位置推定を足して「地図上の本物の現在地」     ← 将来。ここで範囲掃除が本物に
```

## 6. 構成イメージ

```
[スマホ: 部屋スキャン(RoomPlan)] ──→ 間取り
                                      │
                              [Webダッシュボード]  ← 操作 + 地図 + 計器
                                      │ WiFi
                                  [ESP32-CAM]
                                      │ Serial(JSON)
                                  [Arduino UNO] ── モータ/超音波/IMU/電圧
                                      │
                              [クイックルワイパー]
```

## 7. 発表（2026-06-17 テックガレージ）に向けた現実解
- **当日の安全圏**: クイックルを装着 ＋ **既存の障害物回避ファーム**で「部屋を這い回る掃除ロボ」を実機デモ（Web無しでも“自作ルンバ”は成立）。
- **余裕があれば**: Webから起動/停止だけ繋ぐ（最小WebSocket）。
- **当日は見せない（ロードマップとして提示）**: 自作ダッシュボード、部屋スキャン地図、位置推定。
- ストーリー: 「実機で動く現実」＋「Webコックピット＆デジタルツインのビジョン」を分けて語ると映える。位置の不正確さは正直に“将来拡張”と言う方が好印象。
